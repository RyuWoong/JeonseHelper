# User Flows

## Flow 0: Learn Jeonse

```text
전세란?
→ 보증금은 왜 큰가요?
→ 전입신고와 확정일자는 왜 중요한가요?
→ 보증보험은 무엇인가요?
→ 예방 가이드로 이동
```

## Flow 0-1: Prevent Fraud

```text
전세 사기를 어떻게 예방할 수 있을까요?
→ 계약 전 확인
→ 계약 당일 확인
→ 입주 직후 확인
→ 보증보험 가입 가능 여부 확인
→ 위험 신호 발견 시 1번 로드맵으로 이동
```

## Flow 1: Suspected Fraud Roadmap

```text
전세 사기가 의심 돼요
→ 보증보험 여부 확인
   ├─ 가입함: 기관별 이행청구 조건 확인
   ├─ 가입 안 함: 증거 정리, 법원 절차, 지원제도 확인
   └─ 모름: HUG/HF/SGI 등 가입 가능성 조회
→ 집주인 연락 가능 여부 확인
   ├─ 협의 가능: 반환 일정, 서면 증거, 내용증명 검토
   ├─ 지연·회피: 증거 보강, 공식 통지, 지원 상담
   └─ 연락 두절: 신고·법원·지원 절차 검토
→ 계약 상태 확인
   ├─ 만료 전
   ├─ 만료 임박
   └─ 만료 후 미반환
→ 증거 수집
→ 다음 기관 또는 절차로 이동
```

## Flow Principles

- Users should not need to know legal terms before choosing a path.
- Each step should end with concrete next actions.
- Each action should link to required documents, related agencies, and sources.
- Long-running steps should support progress tracking later.
